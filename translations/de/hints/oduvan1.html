Stellen Sie sich vor, Sie müssen zwei Zahlen multiplizieren und das Ergebnis drucken (ausgeben), damit Sie es in der Konsole sehen können. Zum Beispiel: 5 und 6.

print(5*6)

Ausgabe:

Anstelle von 5 und 6 können wir jetzt Variablen verwenden. Für was? Das werden Sie in den weiteren Beispielen sehen.

a = 5
b = 6
print(a*b)

Ausgabe:

30

Sie können die Variable a ändern, das Skript erneut ausführen und das andere Ergebnis sehen

a = 6
b = 6
print(a*b)

Ausgabe:

36

Oder Sie können Ihren Code zweimal kopieren, die Variablen ändern und das Ergebnis von zwei Multiplikationen sehen

a = 5
b = 6
print(a*b)


a = 6
b = 6
print(a*b)

Ausgabe:

30
36

Jetzt kann ich Ihnen zeigen, wie die Funktion den obigen Code vereinfachen kann.

def mult_two(a, b):

    return a*b

print(mult_two(5, 6))
print(mult_two(6, 6))

Ausgabe:

30
36

Wie Sie sehen, enthält die Funktion mult_two nun die gesamte Logik, und Sie ändern nur die Argumente, um unterschiedliche Ergebnisse zu sehen.

Großartig. Vielen Dank, aber können Sie mir bitte die Struktur der Funktion genauer erklären?

Ja, natürlich. Hier ist ein Bild, das die Struktur der Funktion zeigt:

Bei CheckiO beginnt jede Lösung mit einer leeren Funktion. Zum Beispiel:

def mult_two(a: int, b: int) -> int:
# your code here
return 0

... und einem Satz von assert, der Ihre Funktion testet, wenn Sie den Code ausführen.

assert mult_two(3, 2) == 6
assert mult_two(0, 1) == 0

Your goal is to fill an empty function with the correct solution.

Sie beginnt immer mit def.

Der Name der Funktion mult_two - Sie werden den Namen verwenden, um die Funktion in Zukunft aufzurufen

Zwei Argumente (a, b) - das sind Variablen, die ihre Werte erhalten, wenn die Funktion aufgerufen wird. Wenn wir z.B. die Funktion mult_two(5, 6) - innerhalb der Funktion aufrufen, setzen wir die Variable a auf 5 und die Variable b auf 6.

Sie können auch Funktionsdefinitionen mit Typ-Hinweisen sehen.

def mult_two(a: int, b: int) -> int:

    return a*b

Lassen Sie sich nicht von : int und -> int verwirren. Sie erklären dem Benutzer (jemandem, der den Code liest) den erwarteten Typ der Argumente und den Typ des Ergebnisses des Funktionsaufrufs. Aber es ist in Ordnung, diese vorerst zu ignorieren.

Nun sagt der Block der Funktion, der mit vier Leerzeichen und return beginnt, was das Ergebnis des Funktionsaufrufs return a*b sein soll.

In unserem Beispiel print(mult_two(5, 6)) - wird das Ergebnis der Multiplikation der beiden Werte 5 und 6 zurückgegeben und als erstes Argument für die Funktion print verwendet.

Wenn Sie möchten, dass Ihre Funktion mehr als eine Codezeile enthält, sollten alle Zeilen die ersten 4 Leerzeichen enthalten. Lassen Sie uns zum Beispiel eine weitere Variable c verwenden, in die wir das Ergebnis der Multiplikation einfügen und das Ergebnis zurückgeben

def mult_two(a, b):

    c = a*b
    
    return c

Gut. Danke, jetzt ist es klar genug. Aber wenn ich eine korrekte Funktion habe, was soll ich dann tun?

Oben im Code-Editor haben Sie zwei Schaltflächen, "Code ausführen" und "Lösung prüfen".

Wenn Sie die erste drücken, wird der Code aus dem Code-Editor ausgeführt. Wenn Sie auf "Lösung prüfen" klicken, prüft das System Ihre Funktion und zeigt das Feld "Glückwunsch" an, wenn sie wie erwartet funktioniert.

Wenn etwas nicht wie erwartet funktioniert, zeigt Ihnen das Panel den Aufruf der Funktion an, die den falschen Wert zurückgibt

oder wenn es sich um einen Fehler in assert
