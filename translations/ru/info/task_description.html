Как это работает?:

Начальный код твоего решения будет содержать “пустую” функцию (которую нужно наполнить) и набор тестов (asserts) под этой функцией. Это небольшое объяснение разницы. {% endif %}

Asserts которые идут после объявления функции можно использоваться для того, чтобы проверить себя нажатием кнопки Run Code. CheckiO также будет использовать несколько дополнительных тестов для того чтобы проверить решение когда ты нажмешь кнопку Check Solution.

Если решение не прошло тестирование, то на правой панели будет выведено сообщение об ошибке
